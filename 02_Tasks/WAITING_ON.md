# WAITING_ON

## Purpose
Tracks blocked items—tasks stalled because they depend on input, approval, or action from another person. Prevents items from being forgotten and enables proactive follow-ups.

## How to Use
- Add items here the moment they become blocked (don't lose track)
- Check this every Monday and Friday (follow-up days)
- Update "Next Follow-up" column proactively
- Remove item when unblocked and move to appropriate task file
- If waiting > 1 week without response, escalate to manager

## Follow-up Rules
- **After 3 days**: Send reminder message
- **After 5 days**: Escalate to manager if senior person
- **After 1 week**: Status update required (ask for revised timeline)
- **After 2 weeks**: Manager escalation (item removed from blocker list)

## Notes
- Early in ramp: many blockers expected (IT access, training, approvals)
- Be proactive about follow-ups—don't let blockers drift
